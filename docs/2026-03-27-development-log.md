# 个人知识库 — 开发日志

> 日期：2026-03-27
> 开发者：萨娅 × 玉郎

## 项目启动

玉郎提出需求：构建一个 AI 驱动的个人知识管理系统。

### 核心需求
1. 支持文本、PDF、Markdown、图片等多格式输入
2. LLM 按知识类型走不同 Prompt 处理（文言文翻译、成语解释、诗词赏析等）
3. 聚类算法 + 力导向图展示知识关联
4. 集成 Agent 问答能力
5. 答题复习功能

## 技术栈决策过程

### 前端框架
- 对比 Web 应用 vs 桌面应用
- 决定：**Web 应用**（图谱可视化生态最强、跨平台、迭代快）
- 框架：React + TypeScript + Ant Design

### 后端语言
- 对比 Node.js vs Python
- 决定：**Node.js 全栈**（前后端同语言、类型共享、工具链统一）
- 框架：Fastify + TypeScript

### 存储方案
- 决定：**混合存储**（SQLite 存元数据 + 文件系统存原始文档）
- 实际使用：sql.js（纯 JS SQLite，无需编译）

### LLM 接入
- 决定：**调 API**（OpenAI 兼容接口）
- 后扩展为前端可配置多个提供商

## 开发阶段

### Phase 1：脚手架 + 知识输入（15:18 - 15:45）

**搭建内容：**
- pnpm monorepo 结构（client/server/shared）
- SQLite 数据库（5 张表 + migration）
- LLM 适配层（OpenAI adapter，流式输出，重试）
- Prompt 模板引擎（4 种知识类型）
- 知识输入管道（文件上传、PDF 解析、图片 OCR）
- 前端知识输入和列表页面

**遇到的问题：**
- 缺少 uuid 依赖 → 安装解决

### Phase 2：知识图谱可视化（15:53 - 16:28）

**搭建内容：**
- LLM 关联识别（处理新知识时自动找关联）
- 图谱 API（节点+边、聚类、统计）
- Cytoscape.js 力导向图
- 节点按类型着色、按关联数缩放
- 交互：点击查看详情、搜索定位、布局切换

### Phase 3：AI 问答（16:28 - 17:06）

**搭建内容：**
- RAG 引擎（向量嵌入 + 余弦相似度）
- 对话会话管理
- SSE 流式聊天 UI
- 引用来源标注

**遇到的问题：**
- Agent 对话不回复 → 原因：agent 路由使用启动时静态 LLM 实例，未走数据库配置
- 修复：改为动态从 DB 读取活跃提供商（getActiveLLMProvider）

### Phase 4：复习系统（17:17 - 17:46）

**搭建内容：**
- LLM 题目生成（按类型区分题型）
- SM-2 间隔重复算法
- 复习页面（统计面板、答题界面、即时反馈）

**遇到的问题：**
- 复习页面无限加载 → 原因：getTodayStats() 的 while(true) 在无数据时死循环
- 修复：改为有界 for 循环
- 数据库损坏（sql.js 内存溢出）→ 删除重建

### 模型配置功能（15:53 - 16:17）

**搭建内容：**
- ai_providers 表 + CRUD API
- 前端管理页面（添加/编辑/删除/激活/测试）
- 支持 OpenAI/Claude/自定义兼容接口

**遇到的问题：**
- 激活按钮 400 Bad Request → 原因：Fastify 收到 Content-Type: application/json 但 body 为空
- 修复：前端只在有 body 时才设 Content-Type

### 聚类算法（19:17 - 19:30）

**设计方案：**
- 轻量表 clustering_features（只存 keywords/tags/category，不碰原文）
- 混合相似度：0.5×Jaccard(keywords) + 0.4×Jaccard(tags) + 0.1×category匹配
- Louvain 社区检测（纯 TypeScript 实现）
- 手动触发 + 结果缓存 + 参数可调

**为什么选 Louvain：**
- 不需预设聚类数量
- 天然适合图谱场景
- 个人知识库规模秒算

### Token 统计（19:42 - 19:52）

**搭建内容：**
- token_usage 表记录每次 LLM 调用
- 五种场景覆盖：知识处理、关联识别、AI 问答、复习、分类
- 统计页面（今日/本周/本月概览、按模型分布、按用途分布）

### 知识拆分功能（20:17 - 23:32）

**初始版本问题：**
- 拆分过于灵敏（200字就触发）
- 拆分在存入时才执行，预览时看不到
- 无法选择哪些 piece 保存

**迭代版本（v2）：**
- 拆分在预览时执行，同时对每个 piece 做 LLM 解析
- 前端展示勾选卡片，用户可选择保存哪些
- 合并保存时 keywords/tags 取并集，不重复调 LLM
- 已保存变灰，可继续保存剩余

**Prompt 改进：**
- keywords 不再写死数量，改为"根据内容提取适当数量"
- tags 改为"2-5个相关标签"

### 其他 Bug 修复

1. **知识列表弹窗内容溢出** → 弹窗加宽 + body 滚动 + 独立内容区块
2. **图谱删除后报错** → sql.js 不强制外键约束，图谱读取时过滤孤立边，删除时手动级联
3. **KnowledgeInput 中文引号** → JSX 中文引号冲突，改用「」
4. **knowledge.ts 语法错误** → try-catch 结构在编辑时被破坏
